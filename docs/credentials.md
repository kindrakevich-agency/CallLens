# Credentials & API Keys — where to get each one

> **TL;DR — you need nothing to start.** With the defaults (`AI_*_PROVIDER=fake`,
> MinIO for storage), `make up` runs the whole pipeline end-to-end with **no
> external accounts and no paid calls**. You only add real keys when you want a
> real provider.

**Where keys go:** in **dev**, the project root [`.env`](../.env.example) (copied
from `.env.example` by `make init`; it is gitignored — never commit it). In
**production**, inject them via the server environment or the **Symfony secrets
vault** — never in a committed file. After editing `.env`, recreate the affected
container: `docker compose up -d api worker` (and `docker compose restart nginx`).

---

## 1. Secrets you generate yourself (no registration)

These are not external accounts — you create the values.

| Variable | What it is | How to generate |
|---|---|---|
| `APP_SECRET` | Symfony app secret (signs cookies/CSRF). | `openssl rand -hex 32` |
| `JWT_PASSPHRASE` + the JWT keypair | Passphrase protecting the JWT signing keys. | Set any strong passphrase, then run `docker compose exec api php bin/console lexik:jwt:generate-keypair`. Keys land in `apps/api/config/jwt/*.pem` (gitignored). |
| `CUBEJS_API_SECRET` | Signs the Cube analytics token. | `openssl rand -hex 32` |
| `POSTGRES_PASSWORD`, `S3_KEY`/`S3_SECRET` (dev MinIO) | Local infra credentials. | Pick any value (defaults are fine for dev). |
| Webhook **signing secrets** | Per-endpoint HMAC secret for ingestion. | **Auto-generated by the app** — created when you add a webhook endpoint in **Settings → Webhook ingestion** (or via `POST /api/v1/settings/webhooks`). You never set these in `.env`. |

## 2. Deepgram — Speech-to-Text (for `AI_STT_PROVIDER=deepgram`, M3)

Used to transcribe calls with speaker separation.

1. Register at **https://console.deepgram.com/signup** (free credit to start).
2. In the console, open **API Keys** → **Create a New API Key** (give it a name,
   scope *Member*). Copy the key — it is shown once.
3. In `.env`:
   ```dotenv
   AI_STT_PROVIDER=deepgram
   DEEPGRAM_API_KEY=<your-key>
   DEEPGRAM_MODEL=nova-3
   ```
4. Recreate: `docker compose up -d api worker`.

---

## 3. OpenAI — scoring + embeddings (for `AI_LLM_PROVIDER=openai`, `AI_EMBEDDINGS_PROVIDER=openai`, M4/M5)

One OpenAI key covers both LLM scoring and embeddings.

1. Register at **https://platform.openai.com/signup** and add billing
   (**Settings → Billing**) — usage is pay-per-call.
2. Go to **https://platform.openai.com/api-keys** → **Create new secret key**.
   Copy it (shown once).
3. In `.env`:
   ```dotenv
   AI_LLM_PROVIDER=openai
   AI_EMBEDDINGS_PROVIDER=openai
   OPENAI_API_KEY=<your-key>
   OPENAI_LLM_MODEL=gpt-4o-mini
   OPENAI_EMBEDDING_MODEL=text-embedding-3-large
   EMBEDDING_DIM=1024
   ```
4. Recreate: `docker compose up -d api worker`.

> Keep `EMBEDDING_DIM=1024` — it must match the `utterance.embedding vector(1024)`
> column. Changing it requires a new migration.

## 4. Google OAuth — "Sign in with Google" (optional)

Only needed for Google login; email+password works without it.

1. Open **https://console.cloud.google.com/** → create/select a project.
2. **APIs & Services → OAuth consent screen** → configure (External; add your
   email as a test user while in *Testing*).
3. **APIs & Services → Credentials → Create Credentials → OAuth client ID** →
   application type **Web application**.
4. Under **Authorized redirect URIs**, add the callback — it is
   `{APP_URL}/auth/google/check`:
   - Dev: `http://localhost:8081/auth/google/check`
   - Prod: `https://api.calllens.app/auth/google/check`
5. Copy the **Client ID** and **Client secret** into `.env`:
   ```dotenv
   GOOGLE_OAUTH_CLIENT_ID=<client-id>
   GOOGLE_OAUTH_CLIENT_SECRET=<client-secret>
   ```
6. Recreate: `docker compose up -d api` (then `docker compose restart nginx`).

## 5. Object storage — audio files

- **Dev:** **MinIO** runs in the stack — **no account needed**. Defaults
  (`S3_ENDPOINT=http://minio:9000`, `S3_KEY=callens`, `S3_SECRET=callens-secret`,
  `S3_BUCKET=callens-audio`) work out of the box; the bucket is auto-created.
- **Production:** **Hetzner Object Storage** (S3-compatible, EU/GDPR).
  1. In the **Hetzner Cloud Console** (https://console.hetzner.cloud) → **Object
     Storage** → create a bucket (e.g. FSN/NBG/HEL region).
  2. Create **S3 credentials** (access key + secret).
  3. Set in the prod environment:
     ```dotenv
     S3_ENDPOINT=https://<region>.your-objectstorage.com
     S3_REGION=<region>
     S3_BUCKET=calllens-audio
     S3_KEY=<access-key>
     S3_SECRET=<secret-key>
     S3_USE_PATH_STYLE=true
     ```

## 6. Production hosting (M11)

- **Hetzner dedicated server** (AX line) for the app — order in the Hetzner
  Robot/console; install Ubuntu 24.04 + aaPanel + Docker. Used for git-based
  deploy (see [deployment.md](./deployment.md)).
- **Transactional email** (prod) — replace Mailpit with a provider via
  `MAILER_DSN` (e.g. an SMTP provider's DSN). Dev uses Mailpit (no account).

## Quick reference: provider → key → where

| To enable | Set | Key from |
|---|---|---|
| Real transcription | `AI_STT_PROVIDER=deepgram` + `DEEPGRAM_API_KEY` | console.deepgram.com → API Keys |
| Real scoring | `AI_LLM_PROVIDER=openai` + `OPENAI_API_KEY` | platform.openai.com/api-keys |
| Real embeddings/search | `AI_EMBEDDINGS_PROVIDER=openai` + `OPENAI_API_KEY` | platform.openai.com/api-keys |
| Google login | `GOOGLE_OAUTH_CLIENT_ID/SECRET` | console.cloud.google.com → Credentials |
| Prod audio storage | `S3_*` | Hetzner Object Storage |

See [configuration.md](./configuration.md) for the full variable reference.
